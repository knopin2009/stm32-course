## Модуль 1: Основы архитектуры STM32

> **Цель модуля:** студент понимает, как устроен микроконтроллер изнутри, умеет работать с GPIO на уровне регистров, знает систему тактирования и умеет обрабатывать прерывания.

### Лекция 01: Архитектура ARM Cortex-M и семейство STM32

**Что покрывается:**
- MPU vs MCU vs SoC: принципиальные различия, сравнение встраиваемых MPU и MCU, примеры (NXP i.MX6ULL vs STM32F4), правило выбора
- Где микроконтроллеры в повседневной жизни: 9 примеров, общий паттерн «датчики → решение → исполнительные устройства»
- Архитектуры процессоров: ARM, AVR, PIC, RISC-V, x86/x64 — сравнение, разрядность, применение
- RISC vs CISC: различия, примеры кода x86 vs ARM, почему RISC лучше для МК
- История ARM: почему лицензируемая архитектура, а не производство чипов
- Семейства Cortex: A, R, M — принципиальные отличия и области применения
- Линейка Cortex-M: M0+, M3, M4, M7 — сравнение возможностей (Thumb-2, FPU, DSP, кэш)
- Регистровая модель: R0-R15, SP (MSP/PSP), LR, PC, xPSR
- Системные регистры: CONTROL, PRIMASK, BASEPRI, FAULTMASK
- Модель памяти: 4 ГБ адресного пространства, Flash/SRAM/Peripherals
- Таблица векторов прерываний
- Boot-процесс: от Reset до main() (startup, __main, SystemInit)
- Чтение маркировки STM32: STM32F429ZIT6 — расшифровка

**Что НЕ покрывается (будет в следующих лекциях):**
- Детальная регистровая модель ядра (caller/callee-saved, AAPCS, NVIC, приоритеты) — → Лекция 02
- Детальная настройка тактирования (PLL, prescalers) — → Лекция 02
- Конкретная работа с GPIO — → Лекция 03
- NVIC, приоритеты прерываний, вложенность — → Лекция 06

### Лекция 02: Регистровая модель Cortex-M и система тактирования STM32

> **Объединённая лекция:** регистры ядра + RCC. Без регистров ядра непонятно, как работает процессор; без RCC не заработает периферия.

**Часть 1 — Регистровая модель ядра (углублённо):**
- R0-R15: назначение каждого регистра, caller-saved vs callee-saved, AAPCS (передача аргументов, возврат 64-бит, стек для >4 аргументов)
- Стек: MSP vs PSP, рост вниз, bare-metal vs RTOS-задачи
- LR: адрес возврата, вложенные вызовы (PUSH/POP LR), особые значения (0xFFFFFFF9/FD/F1)
- PC: конвейеризация, PC+4 в Thumb-2
- xPSR: APSR (N, Z, C, V флаги → условные переходы), IPSR (номер текущего прерывания), EPSR
- CONTROL: privileged/unprivileged, SPSEL (MSP vs PSP), FPCA
- PRIMASK, BASEPRI, FAULTMASK: три уровня блокировки прерываний, когда использовать каждый
- SysTick: 24-бит таймер ядра, HAL_Delay, RTOS-тик

**Часть 2 — Система тактирования (RCC):**
- Источники: HSI (16 МГц, ±1%), HSE (8 МГц, ±50ppm), LSI (~32 кГц, ±40%), LSE (32768 Гц, ±20ppm)
- PLL: PLLM, PLLN, PLLP, PLLQ — формула расчёта, ограничения (VCO 100-432 МГц, вход 1-2 МГц)
- Шины: AHB (180 МГц), APB1 (45 МГц), APB2 (90 МГц) — делители, периферия на каждой шине
- RCC_EN регистры: AHB1ENR, APB1ENR, APB2ENR — почему без них периферия «мёртвая»
- Тактирование таймеров ×2: почему TIM2 = 90 МГц при APB1 = 45 МГц
- MCO: вывод SYSCLK на PA8/PC9 для отладки
- CSS: защита от отказа HSE → автоматический переход на HSI + NMI
- Flash latency: почему при 180 МГц нужен LATENCY_5
- Полный пример: SystemClock_Config() от HSE до SYSCLK 180 МГц

**Что НЕ покрывается (будет в следующих лекциях):**
- NVIC: приоритеты, вложенность, tail-chaining, группировка preempt/sub — → Лекция 06
- GPIO (только упоминание тактирования) — → Лекция 03
- Таймеры (их тактирование в контексте TIM) — → Лекция 07

**Практическая привязка:** [[02-Практикум/Практика-01]] — настройка SYSCLK 180 МГц, LED на трёх уровнях, MCO

### Лекция 03: GPIO — цифровые входы и выходы

**Что покрывается:**
- Структура пина: MOSFET-пара, защитные диоды, pull-up/pull-down
- Регистры GPIO: MODER, OTYPER, OSPEEDR, PUPDR, IDR, ODR, BSRR, LCKR, AFR
- Режимы: Input, Output, Alternate, Analog
- Типы выходов: Push-Pull vs Open-Drain (когда какой)
- Скорости: Low, Medium, High, Very High — влияние на EMI
- Атомарные операции: BSRR (почему ODR |= не атомарно)
- Блокировка конфигурации: LCKR
- Альтернативные функции: AFR[0], AFR[1] — таблица подключений
- Чтение состояния: IDR (реальный уровень) vs ODR (записанный уровень)
- Токовые ограничения: максимальный ток на пин и на порт
- **Onboard F429I-DISC1:** LED (PG13, PG14), кнопка (PA0)

**Что НЕ покрывается:**
- Прерывания от EXTI — → Лекция 06
- Аналоговые функции (ADC) — → Лекция 09

**Практическая привязка:** [[02-Практикум/Практика-02]] — мигание LED, чтение кнопки, debounce

### Лекция 04: Работа с Flash-памятью

**Что покрывается:**
- Организация Flash: секторы (STM32F429: 4×16 КБ, 1×64 КБ, 7×128 КБ), банки
- FLASH_ACR: кэш инструкций/данных, предвыборка, латентность
- Чтение из Flash: XIP (Execute-In-Place)
- Запись во Flash: unlock → erase → program → lock последовательность
- Ограничения: минимальный размер стирания (сектор), число циклов (~10K)
- Option bytes: защита от записи (RDP), BOR levels, watchdog
- Dual-bank: переключение между банками (OTA-обновления)
- Memory-mapped mode vs programmer mode

**Что НЕ покрывается:**
- EEPROM-эмуляция — упоминается, реализуется в практикуме
- OTA-обновления — упоминается, детально в проектах

**Практическая привязка:** [[02-Практикум/Практика-03]] — сохранение конфигурации LED во Flash, EEPROM-эмуляция

### Лекция 05: HAL и LL библиотеки — философия и практика

**Что покрывается:**
- Уровни абстракции: регистры → LL → HAL
- Архитектура HAL: handle-структуры, callback-модель, state-machines
- LL API: тонкая обёртка над регистрами, zero-overhead
- Инициализация периферии: MX-функции vs ручная настройка
- Error handling: HAL_OK, HAL_ERROR, HAL_TIMEOUT
- Производительность: сравнение HAL vs LL vs регистры (циклы CPU, размер кода)
- Когда что использовать: прототип (HAL) → оптимизация (LL/регистры)
- CubeMX: генерация кода, настройка проекта, ограничения
- Migration: переход от HAL к LL в критичных участках

**Что НЕ покрывается:**
- Конкретная периферия (таймеры, ADC и т.д.) — будет в соответствующих лекциях

**Практическая привязка:** [[02-Практикум/Практика-01]], [[02-Практикум/Практика-02]] — параллельная реализация LED на трёх уровнях

### Лекция 06: Система прерываний (NVIC + EXTI)

**Что покрывается:**
- NVIC: архитектура, приоритеты (4 бита в STM32F4 = 16 уровней)
- Группировка приоритетов: Preempt vs Sub-priority
- Вложенность прерываний: пример с тремя уровнями
- Tail-chaining, Late-arriving: оптимизация NVIC
- EXTI: внешние прерывания (линии 0-15, PVD, RTC, USB, Ethernet)
- SYSCFG: маппинг пинов на EXTI-линии
- PendSV: переключение контекста в RTOS
- Критические секции: PRIMASK vs BASEPRI
- Обработка ошибок: HardFault, MemManage, BusFault, UsageFault
- Debug-обработка HardFault: CFSR, HFSR, MMFAR, BFAR
- **Onboard F429I-DISC1:** кнопка PA0 → EXTI0

**Что НЕ покрывается:**
- SysTick (базовое упоминание) — детально в FreeRTOS — → Лекция 16
- DMA (прерывания DMA) — → Лекция 13

**Практическая привязка:** [[02-Практикум/Практика-03]] — EXTI-кнопка, приоритеты, HardFault-логгер

### Лабораторная 01: Интерактивный LED-контроллер

**Плата:** STM32F429I-DISC1
**Используемая onboard-периферия:** LED (PG13, PG14), кнопка (PA0), LCD TFT (опционально)

**Интеграция тем модуля 1:**
| Тема | Лекция | Компонент на плате |
|---|---|---|
| GPIO | Лекция 03 | LED PG13/PG14, кнопка PA0 |
| EXTI | Лекция 06 | Кнопка PA0 → EXTI0 |
| Тактирование | Лекция 02 | RCC, SYSCLK 180 МГц |
| Flash | Лекция 04 | Сохранение режима LED в Flash |
| HAL + регистры | Лекция 05 | Микс уровней абстракции |

**Задание:** Система управления LED с тремя режимами (ручной, автоматический мигающий, ШИМ-регулировка яркости). Переключение режимов — кнопкой через EXTI. Текущий режим и яркость сохраняются во Flash. Результат отображается на LED + опционально на LCD.

**Критерии оценки:**
- [ ] LED мигает / меняет яркость по кнопке
- [ ] Настройки сохраняются после перезагрузки
- [ ] Использованы EXTI с приоритетами
- [ ] Код работает на HAL и LL (сравнение)

## Модуль 2: Периферия и коммуникация

> **Цель модуля:** студент владеет основными протоколами связи, умеет настраивать таймеры для ШИМ и захвата сигнала, работает с АЦП/ЦАП.

### Лекция 07: Базовые таймеры (TIM2-TIM5, TIM9-TIM14)

**Что покрывается:**
- Архитектура таймера: PSC, ARR, CCR, CNT
- Режимы: Output Compare, PWM, Input Capture, One-Pulse
- PWM Generation: frequency, duty cycle, alignment (edge/center)
- Input Capture: измерение частоты и скважности
- Encoder Interface: чтение квадратурных энкодеров
- Прерывания таймеров: Update, CC, Trigger
- Тактирование таймеров: APB1 vs APB2, умножение при prescaler ≠ 1
- Slave Mode: Trigger, Reset, Gate, External Clock
- Сравнение TIM2-TIM5 (32-bit) vs TIM9-TIM14 (16-bit)

**Что НЕ покрывается:**
- Advanced-таймеры (TIM1/TIM8) — → Лекция 08
- DMA (обновление CCR через DMA) — → Лекция 13

**Практическая привязка:** [[02-Практикум/Практика-04]] — ШИМ LED, секундомер на таймере

### Лекция 08: Продвинутые таймеры (TIM1/TIM8) и мотор-контроль

**Что покрывается:**
- Отличия TIM1/TIM8 от базовых: комплементарные выходы, dead-time
- Dead-time Generator: расчёт для MOSFET/IGBT драйверов
- Break Input: аварийная остановка (overcurrent, overtemperature)
- Repetition Counter: обновление каждые N периодов
- 6-Step PWM: управление BLDC моторами
- Center-Aligned PWM: снижение гармоник
- Synchronization: Master/Slave между таймерами
- Trigger Output (TRGO): запуск ADC по таймеру

**Что НЕ покрывается:**
- Реальный мотор-контроль — упоминается, детально в проектах
- ADC (запуск по TRGO) — → Лекция 09

**Практическая привязка:** [[02-Практикум/Практика-04]] — TIM1 с dead-time, серво-контроль

### Лекция 09: АЦП (ADC) и ЦАП (DAC)

**Что покрывается:**
- ADC архитектура: SAR (Successive Approximation), 12-bit, 2.4 MSPS
- Каналы: регулярные vs инжектированные
- Режимы: Single, Continuous, Scan, Discontinuous
- Sample Time: влияние на точность и скорость
- Аналоговый Watchdog: прерывание при выходе за порог
- Oversampling: увеличение эффективной разрядности (до 16 бит)
- Тактирование ADC: максимальная частота (36 МГц для STM32F4)
- Calibration: factory calibration vs пользовательская
- Внутренний температурный датчик (TS) — канал 16
- DAC: 12-bit, LFSR/Noise wave, DMA trigger
- Buffering: встроенный op-amp, нагрузочная способность

**Что НЕ покрывается:**
- DMA (непрерывный сбор ADC через DMA) — → Лекция 13
- Цифровая фильтрация — упоминается, детально в проектах

**Практическая привязка:** [[02-Практикум/Практика-05]] — внутренний термодатчик, генерация синуса через DAC

### Лекция 10: USART/UART — асинхронная связь

**Что покрывается:**
- Протокол: Start bit, Data bits, Parity, Stop bits
- Baud rate: расчёт из PCLK, допустимая погрешность (< 2%)
- Прерывания: RXNE, TXE, TC, IDLE, ORE
- DMA для USART: circular buffer, без прерываний на каждый байт
- Idle Line Detection: определение конца пакета (Modbus, USB-serial)
- Multi-processor: Address Mark, Idle Line
- Smartcard, IrDA, LIN (специальные режимы)
- Half-Duplex: одноразрядная шина
- Overrun Error: что делать, если не успеваем читать
- USB CDC: виртуальный COM-порт (USB OTG FS на F429I-DISC1)

**Что НЕ покрывается:**
- Протокол Modbus — упоминается, детально в проектах

**Практическая привязка:** [[02-Практикум/Практика-05]] — терминал через USB CDC, отладочные сообщения

### Лекция 11: I2C — двухпроводная шина

**Что покрывается:**
- Протокол: Start, Stop, ACK/NACK, 7-bit/10-bit адресация
- Скорости: Standard (100 кбит), Fast (400 кбит), Fast+ (1 Мбит)
- Роли: Master, Slave, Multi-Master
- Арбитраж: кто проиграл, когда два мастера начинают одновременно
- Clock Stretching: замедление мастера медленным слейвом
- SMBus: отличия от I2C (timeout, PEC, Alert)
- Регистры I2C: CR1, CR2, Timing Register (I2C_TIMINGR)
- DMA для I2C: чтение/запись буферов
- Типичные устройства: датчики, EEPROM, OLED

**Что НЕ покрывается:**
- SPI (параллельная шина) — → Лекция 12

**Практическая привязка:** [[02-Практикум/Практика-06]] — чтение аудио-кодека CS43L22 по I2C (ID, регистры)

### Лекция 12: SPI — высокоскоростная синхронная шина

**Что покрывается:**
- Протокол: MOSI, MISO, SCK, NSS (Full-Duplex)
- Режимы: CPOL/CPHA (0-3), выбор по фронту/сиду
- Скорости: до 42 МГц (APB2/2), ограничение кабеля/платы
- Daisy-Chain: несколько устройств на одной шине
- DMA для SPI: чтение/запись без CPU
- TI mode, Motorola mode: отличия
- CRC: аппаратная проверка целостности
- Типичные устройства: SD-карты, дисплеи, Flash, ADC/DAC
- **Onboard F429I-DISC1:** L3GD20 (гироскоп, SPI), LIS3DSH (акселерометр, SPI), LCD (FSMC)

**Что НЕ покрывается:**
- SD-карты и FatFS — упоминаются, детально в проектах

**Практическая привязка:** [[02-Практикум/Практика-06]] — чтение гироскопа L3GD20 (3 оси), акселерометра LIS3DSH

## Модуль 3: Продвинутые темы

> **Цель модуля:** студент умеет использовать DMA для разгрузки CPU, настраивать RTC для работы с реальным временем, проектировать устройства с низким энергопотреблением.

### Лекция 13: DMA — прямой доступ к памяти

**Что покрывается:**
- Архитектура DMA: 2 контроллера (DMA1, DMA2), 8 потоков каждый
- Stream & Channel: маппинг периферии на каналы
- Режимы: Normal, Circular (для непрерывного чтения)
- Burst Transfer: инкремент адреса, FIFO threshold
- Приоритеты потоков: Low, Medium, High, Very High
- Double Buffering: переключение между буферами без CPU
- Типичные сценарии:
  - ADC → RAM (непрерывный сбор)
  - RAM → USART (передача пакета)
  - RAM → SPI (запись буфера)
  - USART → RAM (приём переменной длины)
  - LCD (FSMC) → DMA (заполнение экрана без CPU)
- Error handling: Transfer Error, Direct Mode Error, FIFO Error
- Когерентность кэша (для M7): Clean/Invalidate D-Cache

**Что НЕ покрывается:**
- Low-Power (режимы сна с DMA) — упоминается, детально в Лекции 15

**Практическая привязка:** [[02-Практикум/Практика-07]] — ADC + DMA (термодатчик), SPI + DMA (гироскоп)

### Лекция 14: IWDG и WWDG — сторожевые таймеры

**Что покрывается:**
- Зачем watchdog: защита от зависания, промышленные стандарты
- IWDG (Independent WDG):
  - Независимый LSI (32 кГц), работает даже в Stop mode
  - Reload ключ: 0xAAAA
  - Window: нет (можно сбросить когда угодно)
  - Настройка предделителя и периода
- WWDG (Window WDG):
  - Привязан к PCLK1 (требует работающего тактирования)
  - Window: сброс только в определённом окне (ранний сброс = ошибка)
  - Прерывание перед сбросом (можно записать лог)
- Концепция «сердца»: задача должна периодически «кормить»
- Мониторинг нескольких задач: каждая кормит свой «канал»
- Что делать при срабатывании: логирование, диагностика
- **Onboard F429I-DISC1:** LED мигание = «жив», остановка = перезапуск

**Что НЕ покрывается:**
- FreeRTOS (watchdog в RTOS) — → Лекция 17

**Практическая привязка:** [[02-Практикум/Практика-07]] — эмуляция зависания (while(1)), восстановление через IWDG

---

### Лекция 15: RTC и Low-Power режимы

**Что покрывается:**
- RTC (Real-Time Clock):
  - LSE (32.768 кГц) — точные часы (внешний кварц на F429I-DISC1)
  - Calendar: Seconds, Minutes, Hours, Date, Month, Year
  - Alarm A/B: прерывание по расписанию
  - Wakeup Timer: периодическое пробуждение из Stop mode
  - Backup Registers: данные, сохраняющиеся при сбросе питания
  - Tamper Detection: обнаружение вскрытия
- Low-Power режимы:
  - Sleep: CPU остановлен, периферия работает (ток ~mA)
  - Stop: всё остановлено, RTC + Backup работают (~µA)
  - Standby: почти всё выключено, только Wakeup/RTC (~nA)
  - Stop с RTC Wakeup: периодическое пробуждение для измерений
- Clock после пробуждения: HSI vs сохранение конфигурации
- Сохранение контекста: что теряется, что сохраняется

**Что НЕ покрывается:**
- FreeRTOS (tickless idle mode) — → Лекция 18

**Практическая привязка:** [[02-Практикум/Практика-08]] — RTC-часы на LCD, пробуждение по Wakeup Timer, LED-индикация

## Модуль 4: RTOS и финальные проекты

> **Цель модуля:** студент понимает концепции real-time ОС, умеет проектировать многозадачные системы, владеет примитивами синхронизации.

### Лекция 16: Введение в FreeRTOS

**Что покрывается:**
- Что такое RTOS: зачем, когда bare-metal недостаточно
- Концепции: задачи, шедулер, приоритеты, квант времени
- Архитектура FreeRTOS: порт для Cortex-M, SVC/PendSV/SysTick
- Создание задач: xTaskCreate, стек, приоритет, handle
- Состояния задач: Running, Ready, Blocked, Suspended
- Шедулер: Preemptive, Cooperative, Time Slicing
- SysTick: тик RTOS (100-1000 ГГц), настройка
- Idle Task: что делает, hook-функция
- Stack Overflow Detection: методы (canary, MPU)
- Порт FreeRTOS на STM32F4: файлы, настройка, Heap_4

**Что НЕ покрывается:**
- Очереди, семафоры, мьютексы — → Лекция 17
- Tickless Idle — → Лекция 18

**Практическая привязка:** [[02-Практикум/Практика-09]] — 3 задачи: LED, гироскоп, LCD

### Лекция 17: Примитивы синхронизации FreeRTOS

**Что покрывается:**
- Queues (очереди):
  - Передача данных между задачами
  - Blocking send/receive: ожидание данных
  - Queue sets: ожидание нескольких очередей
- Semaphores:
  - Binary: синхронизация задач (ISR → Task)
  - Counting: подсчёт ресурсов
- Mutexes:
  - Взаимное исключение (критические секции)
  - Priority Inheritance: решение проблемы инверсии приоритетов
  - Recursive Mutex: повторный захват той же задачей
- Event Groups:
  - Ожидание нескольких событий (AND/OR)
  - Sync Point: все задачи дошли — продолжить вместе
- Critical Sections:
  - taskENTER_CRITICAL / taskEXIT_CRITICAL
  - BASEPRI: маска прерываний
- Timers (Software):
  - One-shot, Auto-reload
  - Callback в контексте Timer Task

**Что НЕ покрывается:**
- Stream Buffers, Message Buffers (FreeRTOS 10+) — упоминаются
- Tickless Idle — → Лекция 18

**Практическая привязка:** [[02-Практикум/Практика-09]] — задача-сенсор → очередь → задача-обработчик → mutex → LCD

---

### Лекция 18: Оптимизация RTOS и Tickless Idle

**Что покрывается:**
- Tickless Idle Mode:
  - Пропуск тиков, когда все задачи Blocked
  - Интеграция с Low-Power (Stop mode)
  - Расчёт времени до следующего события
- Memory Management:
  - Heap_1, Heap_2, Heap_3, Heap_4, Heap_5
  - Fragmentation: причины, обнаружение
  - Static Allocation: без динамической памяти (безопасность)
- Performance Tuning:
  - Оптимальный размер стека (uxTaskGetStackHighWaterMark)
  - Приоритеты задач: баланс responsiveness vs fairness
  - Избегание Priority Inversion
- Debug RTOS:
  - Trace: Percepio Tracealyzer, SystemView
  - Runtime Statistics: CPU usage каждой задачи
  - Stack Overflow: обнаружение до падения
- Safety:
  - MPU (Memory Protection Unit): изоляция задач
  - Assert-макросы: configASSERT
  - Fault-logging в RTOS-контексте

**Что НЕ покрывается:**
- AMP (Asymmetric Multi-Processing) — за рамками курса

**Практическая привязка:** [[02-Практикум/Практика-09]] — Tickless Idle, замер времени сна

### Лабораторная 04: RTOS-система мониторинга с LCD

**Плата:** STM32F429I-DISC1
**Используемая onboard-периферия:** L3GD20 (гироскоп, SPI), LIS3DSH (акселерометр, SPI), LCD TFT, ADC (термодатчик), LED, кнопка

**Интеграция тем модуля 4 + предыдущих:**
| Тема | Лекция | Компонент на плате |
|---|---|---|
| FreeRTOS | Лекции 16-18 | 4 задачи |
| Очереди | Лекция 17 | Сенсоры → обработка |
| Mutex | Лекция 17 | Разделение LCD |
| SPI + DMA | Лекция 13 | L3GD20 + LIS3DSH |
| ADC + DMA | Лекция 13 | Термодатчик |
| Tickless Idle | Лекция 18 | Энергосбережение |
| IWDG | Лекция 14 | Защита от зависания |
| EXTI | Лекция 06 | Кнопка → задача |

**Задание:** Многозадачная система с 4 задачами:
1. **Task_Sensor** (высокий приоритет): чтение гироскопа + акселерометра + температуры по DMA, отправка через очередь

- [ ] Очередь для передачи данных Sensor → Display
- [ ] Mutex для защиты LCD
- [ ] SPI + DMA для сенсоров
- [ ] Tickless Idle (проверка потребления)
- [ ] IWDG «кормится»
- [ ] Кнопка меняет режим через EXTI
